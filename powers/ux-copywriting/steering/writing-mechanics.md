# Writing Mechanics

Source: Amazon Writing Guidelines & AWS Content Design System

## Core Writing Principles

### Use Active Voice
- Active voice is more direct, authoritative, and concise
- Active: "We found page load time impacted conversion..."
- Passive: "We found that conversion has been impacted by page load time..."
- Zombie trick: If you can add "by zombies" after the verb and it makes sense, you're in passive voice
- Passive voice is acceptable when the object is the focus, the actor is unknown, or you want to avoid blaming someone

### Use Simple Words and Be Specific
- Generalizations lead to weak statements. Specific statements with simple words are clear and powerful
- Prefer "It rained every day for a week" to "A period of unfavorable weather set in"

| Complex word | Simpler word |
|---|---|
| advantageous | helpful |
| commence, begin | start |
| commensurate | equal |
| deleterious | harmful |
| disseminate | send |
| endeavor | try |
| erroneous | wrong |
| expeditious | fast |
| facilitate | help |
| inception | start |
| leverage | use |
| obtain | get |
| proficiency | skill |
| promulgate | publish |
| proximity | near |
| subsequently | later |
| utilize | use |

### Remove Indirect Language
- Remove: believe, in general, planning on, should, would like to
- Do: "This is the correct solution."
- Don't: "I believe this is the correct solution."

### Remove Vague Language
- Remove: a number of, complex, generally, often, probably, should
- Replace with data or remove entirely
- Do: "This happens 75% of the time..."
- Don't: "This happens often..."

### Remove Weasel Words
- Words that turn out to be empty upon analysis: should, might, could, often, generally, usually, probably, significant, better, worse, soon, some, most, fewer, faster, slower, higher, lower, many, few
- Replace with data or a statement of intent
- Do: "This will affect 23% of sellers"
- Don't: "A significant proportion of sellers may be affected"

### Remove Adverbs and Adjectives
- In business writing, adjectives and adverbs are a barrier to facts
- Prefer data and actuals over adjectives and adverbs
- Do: "Sales grew 450bps from $XX to $YY in Q4."
- Don't: "Sales grew quickly over the last quarter."

## Tighten Your Prose

Quick fixes:
- "In order to" → "To"
- "Whether or not" → "Whether"
- "Note that" → [Delete entirely]
- "Please" → [Delete entirely]
- "You should" → [Delete entirely]
- "You must" → [Delete entirely] (unless for actual requirements)
- "In the event that" → "if"
- "It is necessary that" → "must" or "should"
- "For the purpose of [-ing verb]" → "to [infinitive verb]"

## Eliminate Fluff

Common fluff to avoid:
- kind of, sort of
- type of
- basically, for all intents and purposes
- definitely
- actually, generally
- individual, specific

Remove words that explain the obvious:
- Don't: "I'm sending this email to you because I want to ask you..."
- Do: Just ask the question
- Don't: "We implemented a successful solution" ("successful" is redundant — a solution by definition solves a problem)

## Avoid Jargon

| Don't use | Use instead |
|---|---|
| Best-in-class / best-of-breed | (Rewrite focusing on what the service can do) |
| below the fold | (Don't use) |
| black-box testing | behavioral testing |
| Disruptive | innovative |
| Ecosystem | environment, system |
| End-to-end | (Avoid. OK in industry terms like "end-to-end encryption") |
| Leverage | use |
| Payload | (Use the specific term, such as "message") |
| Performant | high-performing |
| Public-facing | public |
| Search hits | search results |
| white-box testing | structural testing |
| white-labelled | rebranded |

## Avoid Latinisms

| Latin term | Use instead |
|---|---|
| ad hoc | one-time, when required |
| de facto | actual |
| etc. (et cetera) | and so on |
| ergo | therefore, so, as a result |
| e.g. | for example, such as |
| i.e. | that is, specifically |
| per | for, for each |
| per se | (Do not use) |
| v., vs. | compared to, compared with |
| via | through, by using, with |
| vice versa | the reverse, the other way around |
| viz. | for example, such as |

Exception: "per" is OK in set terms (queries per second, bits per second, on a per-Region basis).

## Positive Language

State clearly what you want to happen using positive language:
- Don't: "We can't continue to show the wrong thing to customers"
- Do: "If we fix these problems, we'll show correct information to customers"
- Don't: "This is not a good use of resources"
- Do: "This is a poor use of resources"
- Don't: "The data does not conclusively show..."
- Do: "The data is inconclusive."

Reserve the negative for denial or antithesis: "We are customer obsessed, not customer driven."

## Redundancies to Avoid

| Redundant | Correct |
|---|---|
| various differences | differences |
| each individual | each or individual |
| reply back | reply |
| future plans | plans |
| end goal | goal |
| final outcome | outcome |
| free gift | gift |
| past experience | experience |
| unexpected surprise | surprise or unexpected |
| sudden crisis | crisis |

## Global English

- Keep sentences under 25 words
- Watch for ambiguous "-ing" and "-ed" words
- Keep adjectives and adverbs close to the words they modify
- Avoid slang, jargon, and colloquialisms
- Use standard terminology and formats consistently
- Use complete sentences
- Use precise words that have only one meaning

### Ambiguous Words to Avoid

| Don't use | When you mean | Use instead |
|---|---|---|
| as | because | because |
| due to | because of | because of |
| just | only | only |
| may | can or might | can or might |
| once | after | after |
| since | because | because |
| while | although or whereas | although or whereas |

## Hyphenation

Hyphenate compound adjectives before a noun:
- high-quality inventory, easy-to-find locations, up-to-date calendar
- closed-door meeting, long-lasting effect, service-linked role
- first-party data, third-party sites, 30-day period

## Plural and Singular Nouns

- Don't use "(s)" or "(es)" — it complicates localization and is ambiguous
- Use the plural form, which can also imply singular
- If necessary, explicitly use both: "Enter the IP address or addresses"
